Problemas identificados

  • Bloqueo de scripts en PowerShell (ExecutionPolicy) al ejecutar .ps1.
  • PATH incompleto (faltaba C:\Windows\System32 y/o Java en PATH) → JMeter no encontraba el JDK.
  • Publicación en GitHub Pages: 404 inicial y build de Jekyll intentando aplicar un tema.

Soluciones propuestas / aplicadas

  • Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass y Unblock-File para ejecutar scripts.
  • PATH saneado en la sesión: C:\Windows\System32; C:\apache-jmeter-5.6.3\bin; %JAVA_HOME%\bin (Java 23).
  • Generación del dashboard: jmeter -g <csv> -o .\docs\jmeter-dashboard.
  • GitHub Pages: branch main, carpeta /docs, y archivo .nojekyll para servir estáticos.

Recomendaciones y siguientes pasos

  • Escalar carga (USERS 10→50→100) y vigilar p95 < 800 ms y errores < 5%.
  • Añadir Assertions (código 200 + validaciones JSON) y revisar keep-alive, compresión y caché.
  • Optimizar endpoints con peores percentiles; evaluar escalado horizontal y repetir pruebas.
